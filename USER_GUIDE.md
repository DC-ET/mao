# Mao 用户手册

本文档面向 Mao 平台的日常使用者与平台管理员，说明如何登录系统、配置模型与 Agent、使用桌面端创建任务、管理工作区、审批工具调用以及排查常见问题。

如需安装、启动或部署系统，请阅读 [README.md](README.md) 与 [DEPLOY.md](DEPLOY.md)。如需了解系统架构与二次开发，请阅读 [docs/technical-design.md](docs/technical-design.md)。

## 1. 产品概览

Mao 是一个企业自托管的 AI Agent 管理与协作平台。平台由三部分组成：

| 模块 | 面向角色 | 主要用途 |
|------|----------|----------|
| 管理后台 | 平台管理员 | 管理用户、角色权限、模型、Agent、Skills、会话、审计与系统配置 |
| 桌面端 Web | 普通用户 | 在浏览器中创建任务、与 Agent 对话、使用云端工作区 |
| Electron 桌面端 | 普通用户 / 研发用户 | 在本地模式下使用 Agent 访问本机目录、执行本地工具和命令 |

Mao 支持两种工具执行模式：

| 执行模式 | 说明 | 适用场景 |
|----------|------|----------|
| CLOUD 云端模式 | 工具在后端服务器上执行 | 统一管控、云端工作区、浏览器访问 |
| LOCAL 本地模式 | 工具通过 Electron 客户端在用户电脑上执行 | 访问本地项目、读取本机文件、执行本地命令 |

## 2. 访问与登录

### 2.1 默认访问地址

本地开发或 Docker Compose 试用环境的默认地址如下：

| 系统 | 地址 | 说明 |
|------|------|------|
| 管理后台 | `http://localhost:5200` | 管理员使用 |
| 桌面端 Web | `http://localhost:5201` | 普通用户使用 |
| 后端 API / Swagger | `http://localhost:9080/api/swagger-ui.html` | 接口调试与查看 |

生产环境地址由部署方配置，请以企业内部发布的访问入口为准。

### 2.2 默认账号

首次启动后，系统会创建默认管理员：

| 字段 | 值 |
|------|----|
| 用户名 | `admin` |
| 密码 | `admin123` |

首次登录后，请立即修改默认密码，并按需创建普通用户账号。

### 2.3 登录方式

Mao 支持以下认证方式，具体可用方式取决于部署配置：

- 本地账号密码登录。
- LDAP 登录。
- 飞书 SSO 登录。

如果登录失败，请先确认账号是否存在、密码是否正确、用户是否被禁用，以及后端服务是否正常运行。

## 3. 管理后台使用说明

管理后台用于平台治理与配置。默认入口为 `http://localhost:5200`，登录后可在左侧菜单进入各功能模块。

### 3.1 数据概览

数据概览用于快速查看平台运行情况，例如用户、Agent、会话或调用相关的统计信息。管理员可以通过该页面了解平台近期使用状态。

建议管理员日常关注：

- 活跃用户数量。
- 会话与任务增长情况。
- 模型调用与用量变化。
- 系统异常或失败趋势。

### 3.2 模型管理

模型管理是使用 Agent 前必须完成的关键配置。Mao 通过 OpenAI 兼容协议调用模型，管理员需要在管理后台配置真实的模型 API 信息。

#### 新增模型

进入「模型管理」，点击新增或添加模型，填写以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 展示给用户看的模型名称 | `GPT-4o`、`DeepSeek V3` |
| 供应商 | 模型服务提供商 | `OpenAI`、`DeepSeek`、`自建网关` |
| 模型标识 | API 请求中使用的模型 ID | `gpt-4o`、`deepseek-chat` |
| API 地址 | OpenAI 兼容接口地址 | `https://api.openai.com/v1` |
| API Key | 模型访问密钥 | 由模型供应商提供 |
| 上下文窗口 | 模型可处理的上下文 token 数 | `128000`、`256000` |
| 支持视觉 | 是否允许发送图片给该模型 | 视觉模型开启 |
| 默认模型 | 新会话是否默认使用该模型 | 按需开启 |

#### 注意事项

- 系统初始化时可能存在占位模型，必须替换为真实 API Key 后才能正常对话。
- `API 地址` 应填写 OpenAI 兼容协议的根地址，通常以 `/v1` 结尾。
- 如果模型不支持图片理解，请不要开启「支持视觉」。
- 建议至少配置一个默认模型，降低普通用户创建任务时的选择成本。

### 3.3 Agent 管理

Agent 是用户在桌面端选择和使用的智能体。管理员可以通过 Agent 管理创建不同职责的 Agent，例如代码助手、运维助手、文档助手、数据分析助手。

#### 创建 Agent

进入「Agent 管理」，点击创建 Agent，填写以下字段：

| 字段 | 说明 |
|------|------|
| 名称 | Agent 展示名称 |
| 描述 | 简要说明 Agent 的用途 |
| 系统提示词 | 定义 Agent 的角色、行为边界和工作方式 |
| 关联 Skills | 指定 Agent 可加载的 Skill 文档；留空时默认加载全部 |
| 标签 | 用于分类、检索和识别 Agent |

#### 系统提示词建议

系统提示词建议包含：

- Agent 的角色定位。
- 可以做什么，不应该做什么。
- 默认输出风格。
- 工具使用原则。
- 安全和审批要求。

示例：

```text
你是企业内部的研发辅助 Agent，擅长阅读 Java、Vue 和 Electron 项目。
你需要先理解用户目标，再检查相关文件，最后给出可执行的修改或说明。
涉及文件写入、命令执行或高风险操作时，应清楚说明目的和影响。
```

### 3.4 Skills 管理

Skills 用于扩展 Agent 的领域知识和工具使用说明。管理员可以在「Skills 管理」中查看和维护平台可用的 Skill 文档。

使用建议：

- 为常见内部系统、研发规范、排障流程编写独立 Skill。
- 保持 Skill 内容清晰、短小、可执行。
- 当某个 Agent 只服务特定场景时，在 Agent 配置中只关联相关 Skills。
- 修改 Skill 后，建议创建一次测试任务确认 Agent 是否按预期使用新能力。

### 3.5 用户管理

用户管理用于维护平台账号。

常见操作：

- 新增用户。
- 编辑用户信息。
- 禁用或启用用户。
- 重置用户密码。
- 查看用户状态。

建议：

- 普通用户只授予必要权限。
- 管理员账号应控制数量。
- 员工离职或不再使用平台时，应及时禁用账号。

### 3.6 角色权限

角色权限用于控制用户可以访问的管理后台功能和业务操作。

常见操作：

- 创建角色。
- 给角色分配权限。
- 将用户关联到合适角色。
- 根据岗位调整权限范围。

权限配置建议：

- 管理员拥有完整配置权限。
- 普通用户不应拥有模型密钥、角色权限、审计配置等敏感管理权限。
- 负责 Agent 配置的成员可以单独授予 Agent 与 Skills 相关权限。
- 定期检查角色权限，避免权限长期堆积。

### 3.7 会话管理

会话管理用于查看用户创建的 Agent 会话与任务记录。

管理员可以通过该功能了解：

- 哪些用户创建了会话。
- 会话使用了哪个 Agent。
- 会话当前状态。
- 会话消息、工具调用和执行结果。

当用户反馈 Agent 异常、回复中断或工具执行失败时，可以优先从会话管理中定位问题。

### 3.8 审计日志

审计日志用于追踪平台关键操作。管理员可用于安全审查、问题排查和合规留痕。

建议重点关注：

- 登录与认证失败记录。
- 用户、角色、权限变更。
- 模型配置变更。
- Agent 与 Skill 配置变更。
- 敏感工具调用和高风险操作。

### 3.9 运行监控

运行监控用于观察平台运行状态。管理员可以结合后端日志、WebSocket 状态、会话状态和工具执行结果判断系统是否健康。

常见排查方向：

- 后端服务是否在线。
- WebSocket 流式连接是否正常。
- Agent 会话是否长时间卡在运行中。
- 工具调用是否频繁失败。

### 3.10 用量分析

用量分析用于观察平台使用趋势和模型消耗情况。

建议关注：

- 不同模型的使用量。
- 不同 Agent 的使用频率。
- 用户或团队的任务量。
- 高消耗任务是否符合预期。

### 3.11 系统设置与通知管理

系统设置用于维护平台级配置。通知管理用于发布或维护面向用户的系统通知。

建议：

- 系统级配置变更前，先确认影响范围。
- 重要维护、停机、模型变更可通过通知管理告知用户。
- 生产环境配置调整应保留操作记录。

## 4. 桌面端使用说明

桌面端是普通用户使用 Mao 的主要入口。用户可以选择 Agent、创建任务、发送消息、查看工具调用、处理审批，以及管理工作区。

### 4.1 进入桌面端

本地默认地址为 `http://localhost:5201`。如果需要使用 LOCAL 本地模式，请运行 Electron 桌面端，而不是只使用浏览器页面。

浏览器 Web 端适合：

- 使用 CLOUD 云端模式。
- 创建云端工作区任务。
- 从 Git 初始化云端工作区。
- 查看会话与 Agent 回复。

Electron 桌面端适合：

- 使用 LOCAL 本地模式。
- 选择本机项目目录。
- 让 Agent 读取或修改本机文件。
- 执行本机命令。

### 4.2 新建任务

点击新建任务后，通常需要完成以下步骤：

1. 选择一个 Agent。
2. 选择执行模式：CLOUD 或 LOCAL。
3. 选择或配置工作区。
4. 输入任务目标。
5. 点击发送或开始。

任务创建后，系统会进入会话页面，Agent 会根据你的输入开始分析、回复或调用工具。

### 4.3 选择 Agent

Agent 由管理员在管理后台配置。不同 Agent 可能拥有不同的系统提示词、Skills 和使用边界。

选择建议：

- 代码修改、项目分析：选择研发或代码类 Agent。
- 文档整理：选择文档类 Agent。
- 运维排查：选择运维类 Agent。
- 通用问答：选择通用助手类 Agent。

如果列表中没有合适的 Agent，请联系管理员创建或调整 Agent 配置。

### 4.4 CLOUD 云端模式

CLOUD 模式下，工具在服务器上执行，工作区也位于服务器配置的工作目录中。

创建 CLOUD 任务时，可选择以下工作区来源：

| 工作区来源 | 说明 |
|------------|------|
| 选择已有工作区 | 使用之前创建过的云端工作区 |
| 创建新工作区 | 创建一个新的云端工作区，可填写项目名，也可留空创建临时工作区 |
| 初始化 Git 工作区 | 通过 HTTPS Git 地址克隆仓库作为工作区 |

使用 Git 初始化工作区时：

- Git 地址必须是 HTTPS 格式，例如 `https://github.com/user/repo.git`。
- 不支持 SSH 格式，例如 `git@github.com:user/repo.git`。
- 私有仓库需要先在「设置 → Git 凭证」中配置对应域名的 Access Token。
- 可以填写分支名；留空时使用仓库默认分支。
- 克隆大型仓库可能需要 1 到 2 分钟。

### 4.5 LOCAL 本地模式

LOCAL 模式下，工具通过 Electron 客户端在你的电脑上执行。该模式可以访问你选择的本地目录，因此更适合处理本机项目。

使用步骤：

1. 启动 Electron 桌面端。
2. 新建任务。
3. 选择 Agent。
4. 选择「本地模式」。
5. 选择本机工作目录。
6. 输入任务并发送。

注意事项：

- 浏览器端不支持 LOCAL 模式。
- Electron 客户端需要保持运行。
- 本地工具可能读取、写入你的工作区文件，或执行本机命令。
- 高风险操作会根据权限档位触发审批。

### 4.6 输入任务

在输入框中清楚描述你的目标。建议包含：

- 你希望 Agent 完成什么。
- 涉及哪个模块、目录或文件。
- 是否允许修改代码。
- 是否需要运行测试。
- 输出形式要求，例如只解释、不改代码、直接修复、给出方案。

示例：

```text
请帮我检查 backend 中用户管理接口的权限控制是否完整。先说明发现的问题，暂时不要修改代码。
```

```text
请修复桌面端新建任务时 Git 地址校验不准确的问题，修改后运行相关前端构建检查。
```

### 4.7 发送图片

如果当前选择的模型开启了「支持视觉」，可以在输入区上传图片。适用场景包括：

- 让 Agent 分析界面截图。
- 根据报错截图排查问题。
- 描述设计稿或页面异常。

如果模型未开启视觉能力，图片可能无法被模型理解。请联系管理员确认模型配置。

### 4.8 选择模型

输入框附近可选择本次任务使用的模型。可选模型来自管理后台「模型管理」。

建议：

- 普通任务使用默认模型。
- 长文档、复杂代码分析选择上下文窗口更大的模型。
- 图片分析选择支持视觉的模型。
- 如果模型请求失败，可切换其他模型重试。

### 4.9 快捷命令与文件引用

桌面端输入框支持快捷命令和文件引用能力，具体可用内容取决于当前工作区、Agent 和 Skills 配置。

使用建议：

- 当任务涉及特定文件时，尽量引用文件或明确路径。
- 当系统提供快捷命令时，优先选择匹配当前任务的命令。
- 如果 Agent 找不到文件，请确认工作区是否正确。

### 4.10 查看回复、思考与工具调用

会话中可能出现以下内容：

| 内容 | 说明 |
|------|------|
| 普通回复 | Agent 给出的自然语言回答 |
| 思考块 | Agent 的阶段性推理或任务规划展示 |
| 工具调用 | Agent 正在读取文件、搜索、执行命令或修改文件 |
| 文件变更 | Agent 对文件的写入或编辑结果 |
| 终端输出 | Shell 命令执行结果 |

建议在 Agent 修改文件或执行命令后，检查工具调用结果和文件变更内容。

### 4.11 停止任务与继续提问

如果 Agent 正在运行，可以点击停止按钮中断当前回复。停止后可以继续发送新消息，让 Agent 基于已有上下文继续处理。

适合停止的情况：

- 任务方向明显不对。
- Agent 正在执行你不希望继续的操作。
- 回复过长或已经得到需要的信息。
- 想补充新的约束条件。

### 4.12 Side Task 并行子任务

平台支持 Side Task 并行子会话能力。用户可以在复杂任务中拆分子任务，让不同会话并行探索或处理不同问题。

使用建议：

- 主会话用于控制总体目标。
- 子任务用于探索独立方向，例如排查一个模块、分析一个报错、验证一种方案。
- 子任务结果需要回到主任务中汇总和确认。

## 5. 权限档位与工具审批

LOCAL 模式下，桌面端提供权限档位，用于控制工具调用是否需要人工审批。

| 权限档位 | 说明 |
|----------|------|
| 只读 | 搜索和读取自动执行，写入和命令需审批 |
| 读写 | 文件读写自动执行，命令执行需审批 |
| 智能审批 | 文件读写自动执行，命令由 AI 判断后自动执行或进入审批 |
| 完全权限 | 所有操作自动执行，无需审批 |

建议默认使用「只读」或「读写」。只有在你完全信任当前任务、工作区和 Agent 行为时，才使用「完全权限」。

### 5.1 审批卡片

当操作需要审批时，桌面端会显示审批卡片。常见审批类型：

- 命令执行待审批。
- 文件写入待审批。
- 文件编辑待审批。

审批卡片通常会展示即将执行的命令或变更描述。你可以：

- 点击「执行」批准操作。
- 点击「拒绝」阻止操作。
- 展开查看完整内容。
- 复制命令或描述进行人工检查。

### 5.2 审批建议

批准前请检查：

- 命令是否在正确目录执行。
- 是否包含删除、覆盖、移动大量文件等高风险行为。
- 是否会访问敏感路径或系统目录。
- 是否会提交、推送或修改远程仓库。
- 是否会安装未知依赖或运行不可信脚本。

如果不确定，应拒绝操作，并要求 Agent 解释目的或提供更安全的替代方案。

## 6. Git 凭证管理

Git 凭证用于 CLOUD 云端模式克隆私有仓库，以及在云端 Shell 中执行需要认证的 Git 远程操作。

入口：桌面端「设置 → Git 凭证」。

### 6.1 新增凭证

点击「新增凭证」，填写：

| 字段 | 说明 |
|------|------|
| 域名 | Git 服务域名，例如 `github.com`、`gitlab.com`、`git.example.com` |
| Access Token | 个人访问令牌 |
| 备注 | 选填，用于识别凭证用途 |

域名可以从完整 URL 中自动规整。例如输入 `https://git.example.com/group/repo.git`，系统会使用 `git.example.com`。

### 6.2 编辑凭证

编辑凭证时：

- 域名不可修改。
- Access Token 留空表示不修改 Token。
- 可更新备注。

### 6.3 删除凭证

删除凭证后，对应域名的私有仓库克隆和远程 Git 操作可能失败。删除前请确认没有正在使用该凭证的任务。

### 6.4 使用限制

- Git 工作区初始化仅支持 HTTPS 地址。
- 私有仓库 Token 需要具备 clone 或读取仓库权限。
- 如果需要 push，还需要写入权限。
- 凭证按完整域名匹配，例如 `git.example.com` 和 `example.com` 不是同一个域名。

## 7. 常见任务示例

### 7.1 让 Agent 分析一个项目

1. 打开桌面端。
2. 新建任务。
3. 选择适合代码分析的 Agent。
4. 选择 CLOUD 或 LOCAL 模式。
5. 选择工作区。
6. 输入：

```text
请先阅读这个项目的 README 和主要目录结构，帮我总结系统架构、启动方式和核心模块。暂时不要修改文件。
```

### 7.2 让 Agent 修改代码

1. 选择正确工作区。
2. 输入明确的修改目标。
3. 要求 Agent 修改后运行检查。
4. 查看文件变更。
5. 审批必要的命令或写入操作。

示例：

```text
请修复用户列表分页查询的问题。修改后运行相关后端测试，并总结改动文件。
```

### 7.3 让 Agent 排查报错

建议提供：

- 报错日志。
- 复现步骤。
- 相关页面或接口。
- 最近改动。

示例：

```text
管理后台登录后跳回登录页。请帮我排查可能原因，重点检查 token 存储、路由守卫和后端认证接口。
```

### 7.4 管理员配置新模型

1. 登录管理后台。
2. 进入「模型管理」。
3. 新增模型。
4. 填写名称、供应商、模型标识、API 地址和 API Key。
5. 按模型能力设置上下文窗口和视觉能力。
6. 保存。
7. 在桌面端创建测试任务验证模型是否可用。

### 7.5 管理员创建专用 Agent

1. 登录管理后台。
2. 进入「Agent 管理」。
3. 创建 Agent。
4. 编写系统提示词。
5. 关联必要 Skills。
6. 添加标签。
7. 保存。
8. 在桌面端选择该 Agent 创建任务测试。

## 8. 安全建议

### 8.1 管理员安全建议

- 首次登录后立即修改默认管理员密码。
- 生产环境必须配置强随机 `JWT_SECRET`。
- 生产环境必须配置 `APP_GIT_CREDENTIAL_SECRET`。
- 不要在文档、截图或聊天中泄露模型 API Key。
- 定期检查用户、角色和权限。
- 定期查看审计日志。
- 对模型、Agent、Skill、系统设置变更建立内部审批流程。

### 8.2 普通用户安全建议

- 不要把敏感密钥、密码、Token 直接发给 Agent。
- LOCAL 模式下谨慎选择工作区，避免选择过大的个人目录或系统目录。
- 审批命令前先阅读命令内容。
- 对删除、覆盖、批量移动、远程推送等操作保持谨慎。
- 任务完成后检查文件变更和测试结果。

### 8.3 Git 凭证安全建议

- 使用最小权限 Token。
- 不要复用个人主账号的高权限 Token。
- Token 泄露后应立即在 Git 平台吊销。
- 离职、权限变更或项目结束后及时删除无用凭证。

## 9. 常见问题

### 9.1 登录失败怎么办？

请检查：

- 用户名和密码是否正确。
- 用户是否被禁用。
- 后端服务是否启动。
- 数据库是否正常。
- 如果使用 LDAP 或飞书 SSO，确认相关配置是否正确。

### 9.2 Agent 没有回复怎么办？

可能原因：

- 未配置可用模型。
- 模型 API Key 无效。
- 模型 API 地址错误。
- 模型服务不可用或超时。
- WebSocket 连接异常。
- 会话处于异常状态。

处理建议：

1. 在管理后台检查模型配置。
2. 查看后端日志。
3. 刷新桌面端页面。
4. 切换其他模型重试。
5. 联系管理员查看会话管理和审计日志。

### 9.3 为什么无法使用 LOCAL 模式？

常见原因：

- 当前使用的是浏览器端，而不是 Electron 桌面端。
- Electron 客户端未启动。
- 本地工作区未选择。
- 本地工具执行被权限策略或审批拒绝。

请使用 Electron 桌面端，并确保已选择正确工作目录。

### 9.4 为什么 Git 克隆失败？

请检查：

- Git 地址是否为 HTTPS 格式。
- 仓库地址是否正确。
- 私有仓库是否已配置 Git 凭证。
- Git 凭证域名是否和仓库域名完全匹配。
- Access Token 是否有读取仓库权限。
- 服务器是否安装 Git。
- 服务器网络是否能访问目标 Git 服务。

### 9.5 为什么上传图片后 Agent 看不懂？

可能原因：

- 当前模型未开启「支持视觉」。
- 模型本身不支持视觉输入。
- 图片过大或格式不受支持。
- Agent 当前任务没有使用图片内容。

请切换到支持视觉的模型，或联系管理员检查模型配置。

### 9.6 为什么工具操作一直要求审批？

这通常与权限档位有关。

- 「只读」模式下，写入和命令都需要审批。
- 「读写」模式下，命令需要审批。
- 「智能审批」会根据风险判断是否审批。
- 「完全权限」通常不会审批，但风险最高。

如果希望减少审批，可以调整权限档位；如果任务涉及敏感文件或命令，建议保留审批。

### 9.7 为什么找不到文件或目录？

请检查：

- 当前任务是否选择了正确工作区。
- CLOUD 和 LOCAL 模式的工作区不是同一个位置。
- Git 工作区是否克隆成功。
- 文件是否被 `.gitignore`、权限或路径问题影响。
- 输入中是否提供了准确路径。

### 9.8 管理后台看不到某个菜单怎么办？

可能原因：

- 当前账号没有对应权限。
- 角色权限未配置。
- 登录状态过期。
- 前端或后端版本不一致。

请联系管理员检查角色权限配置。

## 10. 管理员上线检查清单

正式提供给团队使用前，建议完成以下检查：

- 已修改默认管理员密码。
- 已配置生产环境密钥。
- 已配置至少一个可用模型。
- 已创建普通用户或接入企业登录。
- 已配置合理的角色权限。
- 已创建至少一个可用 Agent。
- 已验证 CLOUD 模式任务可正常创建和回复。
- 如需 LOCAL 模式，已验证 Electron 客户端可连接并执行工具。
- 如需私有仓库，已验证 Git 凭证和 HTTPS clone。
- 已确认审计日志可查看。
- 已向用户说明安全审批和使用边界。
