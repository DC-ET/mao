# 代码审查规范

## 审查流程

1. 理解变更的目的和背景
2. 检查代码逻辑正确性
3. 评估代码质量
4. 验证测试覆盖
5. 提供建设性反馈

## 检查清单

### 正确性
- [ ] 逻辑是否正确
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善
- [ ] 并发安全性

### 可读性
- [ ] 命名是否清晰
- [ ] 函数长度是否合理
- [ ] 注释是否必要且准确
- [ ] 代码结构是否清晰

### 安全性
- [ ] 输入验证
- [ ] SQL 注入防护
- [ ] XSS 防护
- [ ] 权限检查
- [ ] 敏感数据处理

### 性能
- [ ] 算法复杂度
- [ ] 数据库查询优化
- [ ] 缓存使用
- [ ] 内存泄漏风险

### 测试
- [ ] 单元测试覆盖
- [ ] 边界条件测试
- [ ] 异常场景测试

## 反馈原则

- 对事不对人
- 提供具体的改进建议
- 区分必须修改和建议改进
- 肯定好的设计和实现
