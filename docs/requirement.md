# Agent 工作台 - 需求分析文档

> 版本: v0.4 | 更新时间: 2026-05-19

> **文档状态（2026-07）**：本文档为早期需求记录，部分内容与当前实现不一致，请以 [README.md](../README.md) 为准。主要差异：
> - **Agent Hub** 已在 V017 迁移中移除
> - 客户端流式通信使用 **WebSocket**（`/api/ws/stream`），非 SSE 直连前端
> - 桌面端同时支持 **浏览器预览** 与 **Electron** 模式
> - 认证除 LDAP / 飞书外，还支持**本地密码**登录
> - 项目已开源（MIT），定位为**企业自托管**，LLM API Key 由部署方自行配置

## 1. 项目概述

### 1.1 项目定位

构建一个企业级 AI 智能体工作台，类似 OpenAI Codex、WorkBuddy 等产品形态，为企业提供统一的 AI Agent 管理与协作平台。

### 1.2 核心价值

- **统一管理**: 企业在一个平台上管理所有 AI Agent，而非散落在各处
- **权限可控**: 基于角色/岗位的精细权限控制，保障数据安全
- **可审计**: 所有 API 调用经过后端代理，操作全程可追溯
- **灵活扩展**: 支持管理后台配置 Agent，也支持用户自定义搭建

### 1.3 已确认决策

| 决策项 | 结论 |
|--------|------|
| 后端语言 | **Java** |
| 前端框架 | **Vue 3**（管理后台 + 桌面客户端） |
| 桌面客户端框架 | **Electron** + Vue 3（生态成熟、企业级稳定性优先） |
| AI 模型接入 | 管理后台动态配置（baseUrl、apiKey、model 等），不限定具体模型厂商 |
| LLM 通信协议 | **OpenAI 兼容格式**（统一协议层） |
| 部署方式 | **私有化部署** |
| 多租户 | **暂不支持** |
| 客户端形态 | **仅桌面端**（暂不考虑 Web 端） |
| 认证方式 | **LDAP 登录** 或 **飞书扫码登录** |
| 数据库 | **MySQL** |
| Skills 来源 | 管理后台内置 Skills + MCP Server 工具（用户上传能力预留，Phase 2 实现） |
| Hub 范围 | **企业内部** |

## 3. Agent Harness 框架（核心）

> **这是整个项目的重中之重。** 所有 Agent 的运行都基于同一套 Harness 框架。

### 3.1 Harness 定位

Agent Harness 是 Agent 的运行时引擎，负责：
- Agent 生命周期管理（创建、初始化、运行、销毁）
- LLM 调用编排（Prompt 构建、上下文管理、流式输出）
- 工具调度（Skills 调用、MCP 协议通信）
- 会话状态管理（对话历史、上下文窗口）
- 安全沙箱（工具调用权限控制、资源限制）

### 3.2 Harness 核心能力

```
┌──────────────────────────────────────────────────┐
│                  Agent Harness                    │
│                                                  │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────┐ │
│  │  Prompt      │  │  Context    │  │  Safety  │ │
│  │  Engine      │  │  Manager    │  │  Guard   │ │
│  └─────────────┘  └─────────────┘  └──────────┘ │
│                                                  │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────┐ │
│  │  LLM        │  │  Skill      │  │  MCP     │ │
│  │  Adapter    │  │  Dispatcher │  │  Client  │ │
│  └─────────────┘  └─────────────┘  └──────────┘ │
│                                                  │
│  ┌─────────────────────────────────────────────┐ │
│  │            Execution Pipeline               │ │
│  │  (Agent Loop: Think → Act → Observe → ...)  │ │
│  └─────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

| 组件 | 职责 |
|------|------|
| **Prompt Engine** | 根据 Agent 配置（人格、系统提示词、Skills）构建完整的 Prompt |
| **Context Manager** | 管理对话上下文、滑动窗口、摘要压缩 |
| **LLM Adapter** | 统一的 LLM 调用适配层，基于 **OpenAI 兼容协议**对接管理后台配置的各种模型（baseUrl/apiKey/model） |
| **Skill Dispatcher** | 解析 LLM 返回的工具调用意图，调度对应的 Skill 执行 |
| **MCP Client** | MCP 协议客户端，连接外部 MCP Server 获取工具和资源 |
| **Safety Guard** | 工具调用权限校验、内容安全过滤、资源限额控制 |
| **Execution Pipeline** | Agent 核心循环：Think → Act → Observe → 反馈 |

### 3.3 Agent 执行流程

```
用户输入
    │
    ▼
┌──────────────┐
│ Prompt 构建   │ ← 系统提示词 + 人格 + 工具描述 + 对话历史
└──────┬───────┘
       ▼
┌──────────────┐
│ LLM 调用     │ ← 流式请求，经过 LLM Adapter
└──────┬───────┘
       ▼
┌──────────────┐     ┌─────────────┐
│ 响应解析      │────▶│ 工具调用？    │
└──────┬───────┘     └──────┬──────┘
       │                    │ 是
       │ 否                 ▼
       │            ┌──────────────┐
       │            │ Safety Guard │
       │            │ 权限校验      │
       │            └──────┬──────┘
       │                   ▼
       │            ┌──────────────┐
       │            │ Skill/MCP    │
       │            │ 执行          │
       │            └──────┬──────┘
       │                   │
       │                   ▼
       │            ┌──────────────┐
       └────────────│ 结果反馈 LLM  │
                    └──────────────┘
                           │
                           ▼
                    流式输出给用户
```

## 4. 功能模块

### 4.1 Agent 工作台（桌面客户端）

#### 4.1.1 工作台主界面

- 工作台首页展示当前用户可用的 Agent 列表
- 支持按分类/标签筛选 Agent
- 支持搜索 Agent
- 显示 Agent 的基本信息（名称、描述、图标、能力标签）

#### 4.1.2 Agent 交互

- 用户选择 Agent 后进入独立工作空间
- 支持对话式交互（类似 ChatGPT/Codex）
- 支持任务式交互（提交任务、查看进度、获取结果）
- 支持文件上传/下载（Agent 可处理文件）
- 支持 Agent 工作过程的实时流式展示（SSE/WebSocket）
- 支持展示 Agent 的工具调用过程（Skill 调用详情、MCP 工具调用详情）

#### 4.1.3 多会话管理

- 支持同时与多个 Agent 交互
- 会话历史记录查看与恢复
- 会话收藏/置顶

#### 4.1.4 Agent Hub

- 所有用户创建的个人 Agent 可发布到 Hub
- Hub 中展示已发布的 Agent，支持搜索和筛选
- 其他用户可在 Hub 中浏览、安装（添加到自己的工作台）所需的 Agent
- Hub 中的 Agent 显示使用量、评分等信息

#### 4.1.5 个人 Agent 创建

- 用户可通过可视化界面创建个人 Agent
- 配置项包括：
  - **基本信息**: 名称、描述、图标、分类标签
  - **人格设定**: 系统提示词（System Prompt）
  - **Skills 配置**: 选择/组合已有的内置 Skills 工具
  - **MCP 配置**: 连接外部 MCP Server，获取额外工具和资源
  - **模型选择**: 选择使用哪个已配置的 LLM 模型
- 个人 Agent 默认仅自己可见，可选择发布到 Hub

### 4.2 管理后台（Web 端）

#### 4.2.1 用户管理

- 支持 **LDAP 登录** 对接
- 支持 **飞书扫码登录** 对接
- 组织架构管理（部门、团队）
- 角色管理（系统管理员、部门管理员、普通用户等）
- 用户状态管理（启用/禁用/冻结）

#### 4.2.2 模型管理

- AI 模型接入配置（支持多个模型提供商）
  - baseUrl（API 地址）
  - apiKey（密钥）
  - model（模型标识）
  - 其他参数（temperature 上限、max_tokens 等）
- 模型可用性测试（一键验证配置是否正确）
- 模型使用统计与额度管理

#### 4.2.3 Agent 管理

- 管理员创建/编辑/删除企业级 Agent
- Agent 分类与标签管理
- Agent 可见性与权限配置（哪些角色/部门可以使用）
- Agent 运行参数配置（关联模型、Token 限额、并发限制）
- Agent Skills 管理（内置 Skills 注册与配置）
- Agent 使用统计与分析
- Hub 中 Agent 的审核与管理

#### 4.2.4 审计与日志

- API 调用日志（谁、何时、调用了什么、返回了什么）
- Agent 对话日志（完整对话记录可回溯）
- 操作审计日志（管理操作的变更记录）
- 日志检索与导出

#### 4.2.5 系统配置

- 速率限制与配额管理
- 通知配置（飞书机器人等）

### 4.3 服务端（Backend）

#### 4.3.1 核心服务

| 服务 | 职责 |
|------|------|
| **认证服务** | LDAP 对接、飞书 OAuth 对接、Token 管理 |
| **授权服务** | RBAC 权限校验、Agent 访问控制 |
| **Agent Harness** | Agent 运行时引擎（核心），见第 3 章节 |
| **会话服务** | 会话 CRUD、消息存储、上下文管理 |
| **文件服务** | 文件上传/下载、存储管理 |
| **审计服务** | 全链路日志采集、存储、查询 |
| **Hub 服务** | Agent 发布、搜索、安装 |
| **通知服务** | 系统通知、任务完成通知 |

#### 4.3.2 API 设计原则

- RESTful API 为主，WebSocket/SSE 用于实时通信
- 所有 API 必须经过认证（Token/JWT）
- 所有 API 调用必须记录审计日志
- API 版本管理（`/api/v1/...`）
- 统一错误码与响应格式

#### 4.3.3 数据持久化

- 用户数据、权限数据、Agent 配置：**MySQL**
- 对话消息、会话数据：**MySQL**
- 文件存储：对象存储（MinIO / 阿里云 OSS）
- 审计日志：**MySQL**（可选 ELK 辅助检索）

## 5. 非功能需求

### 5.1 安全性

- 全链路 HTTPS 加密
- 密码加密存储（bcrypt/argon2）
- API Token 有效期管理与刷新机制
- 敏感数据脱敏展示
- 操作审计不可篡改
- Agent Skills / MCP 工具调用白名单与权限控制
- LLM API Key 仅在服务端使用，不暴露给客户端

### 5.2 性能

- 桌面客户端启动时间 < 3 秒
- Agent 对话首 Token 响应 < 2 秒
- 管理后台页面加载 < 1 秒
- 支持 1000+ 并发用户（可按需扩展）

### 5.3 可用性

- 服务端可用性 99.9%
- 桌面客户端支持自动更新
- 断线自动重连

### 5.4 兼容性

- 桌面客户端：macOS 12+、Windows 10+
- 管理后台：Chrome、Edge、Firefox 最新两个大版本
