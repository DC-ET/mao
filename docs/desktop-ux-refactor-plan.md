# 桌面客户端 UX 重构方案

> Note: The `bash` tool has been removed. `shell` is now the only command execution tool. See `shell-unification-design.md`.

> 参考 Codex 产品设计思路，对当前 Agent 工作台桌面客户端进行深度重构。

---

## 一、现状问题诊断

### 1.1 信息架构层面

| 问题 | 具体表现 |
|------|----------|
| **产品定位模糊** | 当前是"管理后台 + 聊天窗口"的拼凑体，缺少"委托任务 → 异步执行 → 审查结果"的核心工作流 |
| **导航结构是通用后台模板** | 左侧深色侧栏 + 顶部面包屑是典型的 CRUD 管理后台，不匹配 Agent 工作台的使用场景 |
| **工作台首页信息密度低** | Agent 卡片网格 + 最近对话列表只是"入口聚合"，没有体现任务状态、执行进度等关键信息 |
| **聊天是唯一的交互模式** | 所有操作都回到 ChatView，没有任务面板、执行日志、变更审查等独立视图 |

### 1.2 交互设计层面

| 问题 | 具体表现 |
|------|----------|
| **模式选择打断心流** | 每次点击 Agent 都弹出 CLOUD/LOCAL 选择弹窗，应该记住偏好或在 Agent 配置中预设 |
| **工具执行不可见** | Agent 在后台执行 bash 命令、读写文件，用户在聊天界面只能看到文字流，无法感知执行细节 |
| **权限审批粗糙** | bash 命令只有 Electron 原生弹窗的"拒绝/允许"二选一，缺少"始终允许同类命令"等选项 |
| **没有任务生命周期** | 发送消息后只有"等待中"和"完成"两种状态，缺少排队、执行中、等待审批、失败等中间态 |
| **文件上传体验差** | 只有回形针按钮 + 待上传列表，缺少拖拽上传、文件预览、从 workspace 选择 |

### 1.3 视觉设计层面

| 问题 | 具体表现 |
|------|----------|
| **Element Plus 默认风格** | 全套 Element Plus 组件未经定制，看起来像管理后台而非专业工具 |
| **Markdown 渲染极其简陋** | 只支持粗体、斜体、行内代码，无代码块高亮、无列表、无表格、无链接渲染 |
| **深色模式未真正实现** | CSS 变量定义了 dark 主题但 Element Plus 未适配，大部分组件在深色模式下不可读 |
| **聊天界面布局单调** | 消息气泡只是简单的左/右对齐 + 头像，缺少工具调用卡片、代码变更差异视图等结构化展示 |
| **全局样式有大量残留** | style.css 包含 Vite 模板的 hero、ticks、counter 等无用样式 |

### 1.4 技术架构层面

| 问题 | 具体表现 |
|------|----------|
| **ChatView 过于臃肿** | 661 行的单文件组件，混合了消息渲染、文件上传、WebSocket 管理、SSE 流处理等逻辑 |
| **composables/utils 目录为空** | 没有提取可复用逻辑，所有状态和副作用都在组件内 |
| **Store 层太薄** | 只有 auth store，缺少 session store、agent store、notification store |
| **无离线/缓存策略** | 所有数据实时请求，无本地缓存，断网后完全不可用 |

## 二、Codex 设计理念映射

将 Codex 的 15 个设计要点映射到当前项目的改进方向：

### 2.1 从"聊天"到"委托任务"

**Codex 做法**: 用户提交的不是"一句话"，而是一个"任务"。任务有明确的生命周期（创建 → 执行 → 审查 → 完成）。

**当前差距**: 当前只有一个模糊的 session 概念，用户发消息后不知道任务进展到了哪一步。

**改进方向**:
- 引入 Task 实体：每个用户请求生成一个 Task，包含目标描述、当前状态、执行步骤、产出物
- 工作台首页改造为"任务看板"：按状态分列（排队中 / 执行中 / 等待审批 / 已完成 / 失败）
- 聊天界面不再是唯一入口，任务可以独立查看和管理

### 2.2 围绕真实工作流

**Codex 做法**: 支持读仓库、改文件、跑命令、跑测试、修 bug、生成 PR，AI 在项目环境里工作。

**当前差距**: Agent 执行工具的过程对用户不可见，结果只是文字回复。

**改进方向**:
- 新增"执行过程"面板：展示 Agent 正在读哪个文件、跑什么命令、改了哪些代码
- 新增"变更审查"视图：以 diff 形式展示 Agent 对文件的修改，支持逐文件 accept/reject
- 新增"终端输出"视图：实时展示 bash 命令的 stdout/stderr

### 2.3 异步后台任务

**Codex 做法**: 用户可以把任务交给 Codex 后继续做别的事，适合耗时任务。

**当前差距**: 发送消息后必须等待，没有异步任务管理能力。

**改进方向**:
- 支持"提交任务后离开"：用户可以在等待期间切换到其他 Agent 或页面
- 全局任务状态栏：在侧栏或底部常驻显示正在执行的任务数量和状态
- 任务完成通知：通过 NotificationBell 推送任务完成/失败通知

### 2.4 透明的执行过程

**Codex 做法**: 展示读了什么、改了什么、运行了什么命令，用户能判断方向是否正确。

**当前差距**: ChatView 只有 `content_delta` 文字流和简单的 `tool_call_start/result` 事件。

**改进方向**:
- 将 tool_call 渲染为结构化卡片，而非纯文本：
  - `bash` 命令 → 终端卡片（命令 + 输出 + 状态码）
  - `read_file` → 文件预览卡片（文件名 + 内容片段）
  - `write_file/edit_file` → 差异卡片（before/after diff）
- 每个工具卡片支持展开/折叠，不影响消息流阅读
- 执行时间线：按时间顺序展示所有工具调用，形成可追溯的执行日志

### 2.5 权限与审批机制

**Codex 做法**: 对文件修改、命令执行、网络访问设置可控边界。

**当前差距**: 只有 bash 命令有原生弹窗审批，文件读写无审批，且无策略配置。

**改进方向**:
- 权限策略配置：在设置中支持按操作类型（bash / file_read / file_write / network）设置默认策略（允许 / 每次询问 / 拒绝）
- 命令白名单/黑名单：对 bash 命令支持 pattern 匹配（如 `git *` 始终允许，`rm -rf *` 始终拒绝）
- 审批队列：当多个工具调用排队时，以列表形式展示，支持批量审批

### 2.6 隔离环境

**Codex 做法**: 云端任务运行在独立 sandbox，本地任务有明确工作区。

**当前差距**: workspace 只是一个路径字符串，没有隔离感知。

**改进方向**:
- Workspace 可视化：在侧栏展示当前 workspace 的文件树摘要
- 工作区状态指示：显示当前 workspace 是否有未提交的变更、是否被 Agent 修改
- "先审查再合并"流程：Agent 的修改先存入临时区域，用户确认后才应用到 workspace

### 2.7 适配团队标准

**Codex 做法**: 通过 Skills、项目指令、上下文规则让 Codex 学习团队偏好。

**当前差距**: 创建 Agent 时配置 Skills 和 system prompt，但缺少运行时的规则注入。

**改进方向**:
- 支持项目级 CLAUDE.md / AGENTS.md：Agent 进入 workspace 时自动读取项目规则
- Skills 运行时管理：在聊天界面可以临时启用/禁用 Skills
- 上下文窗口：展示当前 Agent 可见的上下文（项目规则、对话历史、文件列表）

## 三、界面重构方案

### 3.1 整体布局重构

**当前布局**（典型的管理后台）:
```
┌──────────┬──────────────────────────┐
│  Logo    │  面包屑          通知 用户 │
│──────────│──────────────────────────│
│ 工作台    │                          │
│ Agent Hub │      主内容区             │
│ 创建 Agent│                          │
│ 设置      │                          │
└──────────┴──────────────────────────┘
```

**目标布局**（Agent 工作台）:
```
┌────┬─────────────────────────────────────┐
│    │  Agent 名称 / 任务标题    状态  操作  │
│ A  ├─────────────────────────────────────┤
│ G  │                                     │
│ E  │     主内容区（聊天/任务/变更）        │
│ N  │                                     │
│ T  │                                     │
│    ├─────────────────────────────────────┤
│ 侧  │  输入框                    发送 附件 │
│ 栏  ├─────────────────────────────────────┤
│    │  工具执行面板（可折叠）               │
└────┴─────────────────────────────────────┘
```

**关键变化**:
- 侧栏从"导航菜单"变为"Agent 列表 + 会话列表"，类似 VS Code 的侧栏
- 顶部从面包屑变为"当前 Agent/任务上下文栏"
- 底部新增可折叠的"工具执行面板"，展示实时的工具调用日志
- 主内容区根据场景切换：聊天视图 / 任务看板 / 变更审查

### 3.2 聊天界面重构

**当前问题**: ChatView.vue 661 行单文件，功能混杂，消息渲染简陋。

**目标架构**:

```
ChatView.vue (容器，~150 行)
├── ChatHeader.vue         — Agent 信息 + 状态 + 操作按钮
├── MessageList.vue        — 消息列表容器，管理滚动和加载
│   ├── MessageBubble.vue  — 单条消息（文本 + 元数据）
│   ├── ToolCallCard.vue   — 工具调用卡片（bash/file/diff）
│   ├── CodeBlock.vue      — 代码块（语法高亮 + 复制）
│   └── DiffView.vue       — 文件变更差异视图
├── ChatInput.vue          — 输入框 + 文件上传 + 快捷操作
├── ToolExecutionPanel.vue — 底部可折叠的工具执行日志
└── WorkspaceBar.vue       — LOCAL 模式下的 workspace 选择器
```

**消息渲染升级**:

| 当前 | 目标 |
|------|------|
| 粗体/斜体/行内代码（正则替换） | 完整 Markdown 渲染（引入 marked + highlight.js） |
| tool_call 显示为纯文本 | 结构化工具卡片，带图标、状态、可展开详情 |
| 无代码块支持 | 代码块带语法高亮、行号、一键复制 |
| 无差异视图 | 文件修改以 unified diff 格式展示，支持 accept/reject |
| 消息只有头像 + 文字 + 时间 | 包含执行状态（成功/失败/进行中）、耗时、token 消耗 |

**工具调用卡片设计**:

```┌─────────────────────────────────────────┐
│ ▶ bash                          2.3s  ✓ │
│   $ npm run test                        │
│   ─────────────────────────────────────  │
│   PASS src/utils.test.ts                │
│   Tests: 12 passed, 12 total            │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ ▶ edit_file                       0.1s  │
│   src/components/Button.vue             │
│   ─────────────────────────────────────  │
│   - const props = defineProps<{         │
│   + const props = withDefaults(         │
│   +   defineProps<{                      │
│   +     variant?: 'primary' | 'ghost'   │
│   +   }>(),                             │
│   +   { variant: 'primary' }            │
│   + )                                   │
│   [Accept]  [Reject]                    │
└─────────────────────────────────────────┘
```

### 3.3 工作台首页重构

**当前**: Agent 卡片网格 + 最近对话列表，纯静态展示。

**目标**: 任务驱动的工作台，一眼看到所有进行中的工作。

```
┌─────────────────────────────────────────────────┐
│  工作台                                          │
│                                                  │
│  ┌─ 进行中 ─────────────────────────────────────┐│
│  │ 🔵 重构登录模块 (GPT-4)     执行中  3/7 步骤  ││
│  │ 🔵 修复 CI 失败 (Claude)    等待审批           ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  ┌─ 最近完成 ───────────────────────────────────┐│
│  │ ✅ 添加单元测试 (GPT-4)     2小时前  [查看]    ││
│  │ ✅ 代码审查 (Claude)         昨天    [查看]    ││
│  │ ❌ 优化查询性能 (GPT-4)     昨天    [重试]    ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  ┌─ 我的 Agent ─────────────────────────────────┐│
│  │  [代码助手]  [文档写手]  [测试工程师]  [+ 新建] ││
│  └──────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

**关键变化**:
- 首屏展示"进行中的任务"，而非 Agent 列表
- 每个任务卡片显示：Agent 名称、任务摘要、当前步骤、状态、耗时
- 点击任务卡片直接进入对应的聊天/任务详情
- Agent 列表降级为次要信息，放在下方

### 3.4 Agent Hub 重构

**当前**: 卡片网格 + 详情弹窗，类似应用商店。

**优化方向**:
- 卡片信息层次优化：突出 Agent 能力描述，弱化评分/安装量等次要信息
- 详情页从弹窗升级为独立页面或侧滑面板，展示完整信息（示例对话、能力列表、配置说明）
- 支持"一键试用"：不安装也能临时体验
- 搜索结果支持按能力筛选（如"擅长代码审查"、"支持本地执行"）

### 3.5 设置页重构

**当前**: 两个 tab（通用 + 关于），功能单一。

**目标**: 按功能域分组，增加关键配置项。

```
设置
├── 通用
│   ├── 服务器地址
│   ├── 主题（浅色/深色/跟随系统）← 真正实现深色模式
│   └── 语言
├── 权限与安全
│   ├── Bash 命令策略（允许/询问/拒绝）
│   ├── 文件操作策略
│   ├── 命令白名单/黑名单
│   └── 网络请求策略
├── 执行环境
│   ├── 默认执行模式（CLOUD/LOCAL）
│   ├── 默认 Workspace 路径
│   └── 工具执行超时时间
├── 通知
│   ├── 任务完成通知
│   ├── 审批请求通知
│   └── 通知方式（系统通知/应用内）
└── 关于
```

## 四、技术重构方案

### 4.1 组件拆分

**ChatView.vue 拆分清单**:

| 新组件 | 职责 | 预估行数 |
|--------|------|----------|
| `ChatView.vue` | 容器，组合子组件，管理会话生命周期 | ~150 |
| `ChatHeader.vue` | Agent 信息、模式状态、操作按钮 | ~80 |
| `MessageList.vue` | 消息列表、滚动管理、历史加载 | ~120 |
| `MessageBubble.vue` | 单条消息渲染（文本 + 元数据） | ~100 |
| `ToolCallCard.vue` | 工具调用的结构化展示 | ~150 |
| `CodeBlock.vue` | 代码块（语法高亮 + 复制） | ~60 |
| `DiffView.vue` | 文件变更的 diff 视图 | ~120 |
| `ChatInput.vue` | 输入框、文件上传、快捷操作 | ~100 |
| `ToolExecutionPanel.vue` | 底部工具执行日志面板 | ~100 |
| `WorkspaceBar.vue` | Workspace 选择和状态显示 | ~60 |

### 4.2 Composables 提取

| Composable | 职责 |
|------------|------|
| `useChat` | 管理消息列表、发送消息、处理 SSE 流 |
| `useToolExecution` | 管理工具调用状态、审批队列、执行日志 |
| `useWorkspace` | 管理 workspace 路径、文件列表、变更状态 |
| `useSession` | 管理会话创建、恢复、切换 |
| `useMarkdown` | Markdown 渲染（封装 marked + highlight.js） |
| `useSSE` | SSE 连接管理、事件解析、重连逻辑 |
| `useLocalTool` | WebSocket 连接管理、本地工具执行 |
| `useNotification` | 通知轮询、未读计数、标记已读 |

### 4.3 Store 层补充

| Store | 状态 |
|-------|------|
| `auth` (已有) | token, user, login/logout |
| `session` (新增) | sessions[], currentSession, loading |
| `agent` (新增) | agents[], currentAgent, hubAgents |
| `task` (新增) | activeTasks[], taskStatus, executionLog |
| `notification` (新增) | unreadCount, notifications[] |
| `settings` (新增) | theme, language, permissions, defaults |

### 4.4 依赖升级

| 变更 | 原因 |
|------|------|
| 新增 `marked` | 完整 Markdown 渲染 |
| 新增 `highlight.js` | 代码语法高亮 |
| 新增 `@vueuse/core` | 通用 composables（useDark, useLocalStorage, useResizeObserver 等） |
| 新增 `diff` 或 `diff2html` | 文件变更差异渲染 |
| Element Plus 按需引入 | 减少打包体积，避免全局注册所有图标 |
| 移除无用资产 | hero.png, vite.svg, vue.svg, HelloWorld.vue |

### 4.5 样式体系重构

**清理**:
- 删除 style.css 中的 Vite 模板残留样式
- 删除 HelloWorld.vue

**引入设计系统**:
- 基于 CSS 自定义属性建立统一的设计 token（颜色、间距、圆角、阴影）
- Element Plus 主题定制：覆盖默认紫色为产品主色调
- 深色模式完整实现：Element Plus dark CSS + 自定义组件暗色适配
- 响应式布局：支持窗口缩小到 800px 时侧栏自动折叠

## 五、实施优先级

### P0 — 核心体验（必须先做）

1. **Markdown 渲染升级** — 引入 marked + highlight.js，替换正则渲染
2. **ChatView 组件拆分** — 从 661 行单文件拆分为 10 个子组件
3. **工具调用卡片** — 将 tool_call 渲染为结构化卡片（bash 终端 + 文件预览）
4. **清理全局样式** — 删除 Vite 残留，建立 CSS 变量体系
5. **深色模式适配** — Element Plus dark 主题 + 自定义组件暗色

### P1 — 交互升级

6. **Composables 提取** — useChat, useToolExecution, useSSE 等
7. **Store 层补充** — session, agent, task stores
8. **权限策略配置** — 设置页新增权限配置项
9. **模式选择优化** — 记住用户偏好，去掉每次弹窗
10. **任务状态展示** — 工作台首页展示进行中的任务

### P2 — 体验打磨

11. **文件变更 diff 视图** — 展示 Agent 的文件修改
12. **工具执行面板** — 底部可折叠的执行日志
13. **Workspace 可视化** — 侧栏展示 workspace 文件树
14. **全局任务状态栏** — 侧栏底部显示活跃任务
15. **Agent Hub 优化** — 信息层次、一键试用

### P3 — 高级功能

16. **异步任务管理** — 提交任务后离开、任务看板
17. **多任务并行** — 同时运行多个 Agent 任务
18. **通知系统升级** — WebSocket 实时推送
19. **离线缓存** — 本地缓存会话和 Agent 数据
20. **项目规则注入** — 自动读取 workspace 中的 AGENTS.md

## 六、设计原则总结

1. **任务优先，而非聊天优先** — 首页展示的是"正在做什么"，而非"和谁聊天"
2. **过程可见，而非只看结果** — 工具调用、文件变更、命令输出都要可视化
